Getting Started¶

This software communicates with XBee through the Java RXTX serial library. That said, it would typically run on a PC and serve as the Computer<-> XBee gateway. This is useful if you want to doing things like provide internet access to your XBee network (e.g. query XBee from internet), but possibly you just want to log your XBee data to a local file or database. I'm not aware of any cost effective Java microcontrollers that could run this software, but hopefully there will be one available at some point in the future.

If you are looking for an XBee library that runs on a microcontroller, I have recently released xbee-arduino, a port of XBee API for the Arduino.

If you don't have XBee hardware, here's a wiki to help you decide which type to get ChoosingAnXBee

There are several ways to connect an XBee to your computer but I'm going to offer two:

  1. XBee Explorer USB
  2. Unchipped Arduino + XBee Shield

The first option is the most cost effective but the second option can make sense if you have a idle Arduino handy and XBee Shield. With the Arduino option, you must first remove your Atmega chip. Of course when you remove the Atmega, it can't run Arduino code and you are only using it for USB serial and power.

Note: You don't need the expensive Digi development kit to get started your XBee project; besides it requires a RS-232 serial port, which fewer computers have these days.

Option 1 (XBee Explorer USB):

  1. Plug your XBee into the XBee Explorer USB socket, and optionally into a breadboard (if you want to access I/0). You're done!. Although you may want to connect an LED to the XBee "Associate" pin.

Option 2 (Unchipped Arduino):

  1. Remove the Atmega chip from the Arduino (hint: it's the biggest component on the Arduino). I had a hard time removing it without causing damage. I ended up using a tiny screwdriver to carefully pry it out. Note the orientation of the chip (dot should be closest to the edge of the board) for when you replace it.
  2. Next place both USB/XBee jumpers in the USB position.
  3. Connect your shield, drop in your XBee and you are ready to go.

Both the Arduino and XBee Explorer USB use the FTDI USB/Serial chip. FTDI enables a virtual serial port over USB and provides free drivers http://www.ftdichip.com/FTDrivers.htm for nearly all platforms (win/mac/linux etc.). If you have installed Arduino software and successfully communicated with Arduino, you already have the drivers.

Parts¶

XBee Explorer USB¶

  • XBee Explorer USB This is the easiest device for connecting your XBee to your computer. You will need to solder headers to the board if you want to access Reset or I/O pins.
  • Mini USB Cable

Remote XBee¶

If you just want the bare minimum parts to power up a remote XBee, this is all you should need.

  • XBee Breakout Board Kit (SparkFun has one too)
  • 3.3 Voltage Regulator + Capacitors (Kit)
  • Breadboard
  • Battery Pack or Power Supply
  • Hookup wire

Note: If you don't want to mess with voltage regulators and capacitors, SparkFun has a 5/3.3V breadboard power supply kit. This was my first soldering project.

Arduino XBee Shield¶

  • Arduino
  • Aduino XBee Shield
    • Kit
    Or
    • Assembled
  • USB Cable -- You probably have one lying around somewhere.

Updating Firmware¶

One of the first things you should do when you get your XBee is to check the firmware. All of the XBees I have received were shipped with old firmware. As of this writing, I recommend the following firmware versions for use with this software: series 1 (10C8), series 2 ZB Pro (2.X.4.1). Note: you need Windows to run the X-CTU software for firmware upgrades. Be aware that you can't upgrade your XBee with a Unchipped Arduino XBee Shield because it does not provide RTS/CTS. For this reason I recommend the XBee Explorer for firmware upgrades.
